What Changed

Problem

The maps service relied on external APIs (OpenStreetMap Overpass) for finding emergency services. This was slow, unreliable, and had no trauma capability data. There was no Golden Hour tracking in dispatch messages.

Solution

Replaced external API calls with a local JSON data file containing 28 verified emergency service records (8 hospitals + 3 police + 3 ambulance per city) for Bhubaneswar and Chennai. Added trauma level classification, Golden Hour countdown, and formatted dispatch messages.

Files Changed

  • NEW data/hospitals.json — 28 records: real hospitals, police stations, ambulance contacts for Bhubaneswar & Chennai
  • MODIFIED app/services/maps.py — Rewrote to use local JSON + haversine. Removed Overpass API. Geoapify as commented alternative.
  • MODIFIED app/services/triage.py — Added format_golden_hour(), format_dispatch_message(). Updated dispatch flow.
  • MODIFIED app/models/schemas.py — Added trauma_level, is_24x7 to EmergencyService. Added triage_started_at to SessionState.
  • MODIFIED app/config.py — Added hospital_data_path setting.
  • MODIFIED prompts/triage_system.txt — Updated STATE 4 (DISPATCH) instructions for new format.
  • NEW tests/test_maps.py — 24 tests for data loading, haversine, city detection, trauma classification, service lookup
